# Ensō — the hands-on walkthrough

Follow this top to bottom. Every command and prompt is **copy-paste ready**. The arc is *vibe until it hurts,
then earn the discipline* — ten moves, each one git tag you can check out and diff.

> `$ …` = a shell command · `› …` = type this into the open `claude` session · `# …` = a comment.

## Prerequisites
- **Node 18+** — `node -v`
- **The `claude` CLI, logged in** — run `claude` once, `/login` if needed.
- *(optional)* **Playwright** — only for the stage-8 smoke: `npm i -D playwright && npx playwright install chromium`.

## Setup

The stage tags live in `.built/` (generated by the build script), **not** in the source folder you clone — so
generate them once, then `cd` in. Every `git checkout <stage>` / `git diff <stage>` below runs inside `.built`.

```bash
git clone <the-enso-repo> enso && cd enso
bash course/build-history.sh .built    # creates the ten stage tags into ./.built
cd .built                              # the cold-open + every checkout/diff below run HERE
git config core.hooksPath .githooks    # arm the floor (runs on every commit)
git checkout 1-vibe                    # the baseline — build forward from here
claude                                 # one session, kept open for the whole build
```
> Want to build every file from an empty folder with no tags at all? Follow **`BUILD.md`** instead.
> (If `git checkout 1-vibe` says *pathspec did not match*, you're in the source folder — run the build script and `cd .built`.)

## The cold open (show this FIRST)
Before you teach anything, show the payoff on the finished reference:

```bash
git checkout 9-reference && node server.mjs    # http://localhost:4173
# type "add a theme picker" → Enter → drag the card to Doing → watch it build, gate, ship.
```
> **Say:** "That board just wrote its own feature. Now let me show you how to build the whole thing from an empty file." Then `git checkout 1-vibe` and begin.

› build a minimal single-file todo: index.html, vanilla JS, localStorage. works by opening the file.

**See:** three columns; drag a card; it sticks. Remember `COLUMNS` — the gate checks it soon.

## Act II — the turn: hand it to an agent, then earn each discipline

### 4 · `4-agentic` — hand it to an agent (MCP hands)
This adds the backend, and swaps `index.html` to the **reference UI in final form** (the agent feed + board
chrome) — the stable surface every later stage builds discipline *around*; we don't re-vibe the frontend after
this, so it stays byte-stable through `9-reference`. Build it in one session prompt, then commit the backend by hand:
> Want to *derive* it yourself instead of one mega-prompt? **`BUILD.md` breaks this stage into four small,
> verifiable steps (4c-i…iv)** — the only place the two guides differ on purpose.
```
› add board-mcp.mjs (stdio JSON-RPC: list_cards/add_card/move_card/seed/clean; board.json is the store) + .mcp.json. add server.mjs: serve the board on :4173; on POST /build spawn `claude -p --output-format stream-json --verbose --mcp-config .mcp.json --dangerously-skip-permissions` to implement one card, stream it over SSE; move the card to Done via board.move_card. add report.mjs. follow the enso-design skill.
```
```bash
$ git add board-mcp.mjs .mcp.json server.mjs report.mjs && git commit -m "agentic"
$ node server.mjs        # drag a card to Doing — the agent runs
```
**See:** reasoning + tool calls stream; the card moves itself via `board.move_card`. **And the question lands: how do I trust this?**

### 5 · `5-gate` — the gate (commit floor, part 1) ★
```
› add gate.mjs — a deterministic Node check (7 checks): extract index.html's inline <script> and syntax-check with new Function; assert NO emoji, NO raw × glyph, NO pure #fff/#000 (use the --bg/--fg tokens), the --accent token is defined, the COLUMNS array declares todo/doing/done, and lint report.mjs the same way (a rendered surface; a check in a try fails open if you forget it). exit 1 with the violations. add .githooks/pre-commit running `node gate.mjs || exit 1`; chmod +x it.
```
Prove it — the hero beat:
```bash
$ node gate.mjs                      # ✓ gate passed
# add an emoji to index.html, then:
$ git add -A && git commit -m "x"    # ✗ GATE BLOCKED — emoji found. the leash held.
```
**The point:** a machine — not the author — says no. *That's* why you can hand an agent the keyboard. (It catches slop, not "wrong"; a local hook is advisory — the same check in CI is the wall.)

### 6 · `6-tests` — tests (commit floor, part 2) ★
```
› add a test/ suite using Node's built-in runner (node --test, no Jest): the gate blocks emoji and a missing COLUMNS; the board tools add/list/move cards; persistence survives. wire .githooks/pre-commit to run `node gate.mjs && node --test`.
```
```bash
$ node --test            # all green. break persistence → red → the commit is blocked.
```
**The point:** the gate proves it's *clean*; the tests prove it *works*. Now the floor is complete.

### 7 · `7-spec` — write down what you mean (the punchline)
```
› write spec.md (user stories, acceptance scenarios = the tests, success criteria, non-goals), constitution.md (the engineering rules the reviewer audits), plan.md, and tasks.md (the board's cards derive from here).
```
**The point:** you built a working, gated, tested loop in 4–6 with **no spec** — so autonomy forced the **floor** (gate + tests), not a spec. Stage 7 is realizing that floor was already an implicit spec, and making it explicit and human-owned. Spec-driven, *discovered* — and your own `git log` proves the spec arrived last.

### 8 · `8-release-gate` — the audit (smoke → receipt)
```
› add qa.mjs — one thin Playwright smoke (page loads, columns render, the feature toggles → screenshot + .receipts/<id>.qa.json). add review.mjs — a SEPARATE `claude -p` that audits the diff against spec.md + constitution.md and the smoke result, writing a .receipts/<id>.review.json bound to the diff hash. wire both into server.mjs AFTER the commit floor: they gate PROMOTION to Done, never the commit.
```
**The point:** generator ≠ evaluator, and a claim isn't proof — judgment must leave a **receipt** bound to the diff. The review is a signal, the smoke is rendering, **the floor is the proof**.

### 9 · `9-reference` — the ultimate test, live ★
```bash
$ node server.mjs        # add "a theme picker" → drag to Doing → watch the whole loop ship it
```
**The point:** the cold-open moment again — but now you can name every safeguard between the agent and main.

### 10 · `explore-conductor` — the circle opens (take-home)
```bash
$ node scripts/conduct.mjs    # the loop reads the board and builds the next task itself
```
**The point:** you stopped writing the code, then stopped typing the prompt — and never stopped owning the spine.

## Troubleshooting
- **Gate fails with `column "todo" not declared in COLUMNS`** — you named the array `COLS`/inlined it. Name it exactly `COLUMNS` (stage 3).
- **`--mcp-config` errors at build time** — that flag belongs to the child the *server* spawns at runtime, not your build session. `.mcp.json` doesn't exist until stage 4.
- **The commit isn't blocked** — arm the hook: `git config core.hooksPath .githooks && chmod +x .githooks/*`.
- **`server.mjs` crashes on start** — it imports `report.mjs` and `board-mcp.mjs`; both are part of stage 4.
